Jekyll

jekyll 是什么？以下是转自 http://jekyll.com.cn/docs/home 对 jekyll 的介绍。

Jekyll 是一个简单的博客形态的静态站点生产机器。它有一个模版目录，其中包含原始文本格式的文档，通过 Markdown （或者 Textile） 以及 Liquid 转化成一个完整的可发布的静态网站，你可以发布在任何你喜爱的服务器上。

jekyll目录结构一般如下：

  • _config.yml 是配置文件。关于配置项详细文档见 jekyll配置。
  • _drafts 该目录下的文件是未发布的文章，这些文件都没有title.MARKUP数据。
  • _includes 该目录下的文件可以被其他文章引用。
  • _layouts 该目录下的文件是模板。
  • _posts 该目录下存放文章。文件名必须符合YEAR-MONTH-DAY-title.MARKUP格式。
  • _site jekyll将转化结果存入该目录下。
  • index.html 是博客主页。

jekyll 已经帮我们做了非常多的事，我们需要做的就是配置 _config.yml 文件、在 _posts 目录下写文章。你也可以修改 css 文件，js 文件和 HTML 文件实现自己的个性化网页设计。

jekyll支持 Markdown 和 Textile 两种标记语言格式，笔者使用的是 Markdown，所以接下来说说 Markdown。

Markdown

Markdown 语法由符号组成，就可读性而言与 Textile 相比就见仁见智了，个人觉得前者的可读性会更高。这两者都是支持HTML的，所以，相对复杂的内容编辑就需要一些HTML基础。

Markdown 语法很简单，具体请参考 Markdown。
