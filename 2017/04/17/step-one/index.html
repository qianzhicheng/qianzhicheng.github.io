本系列文章旨在于分享自己在 GitHub 上创建 blog 的经验，东西不多，也没有什么高深难懂的技术点，毕竟 jekyll 就是为了避免过多的代码编写，让注意力更多的集中于网页内容。

GitHub&GitHub Pages

GitHub 是一个具有版本管理功能的代码仓库。Github Pages 是 Github 提供的静态页面托管服务，它可以用来做项目主页或者个人主页。GitHub 是鼓励用户用 Github Pages 搭建个人博客的。https://github.com/blog/272-github-pages

Create a blog and spread your ideas. Whatever you want!

创建项目

首先注册一个 GitHub 账号，登录后点击 New Repository，进入创建项目仓库页面

注意：项目名称必须是 username.github.io 格式。

安装Git

Windows 下的 git 安装还是比较简单的，在 git for windows 上下载安装包直接安装就行。

配置Git

首先在本地创建 ssh key

your_email 为注册 GitHub 账号的邮箱。然后一路回车全部默认就行。之后会在当前文件夹生成一个 .ssh 文件夹，进入 .ssh，打开 id_rsa.pub，复制里面的 key。

回到 GitHub，进入 settings 界面，点击 SSH and GPG keys -> New SHH key,填入 Title，把刚刚复制的 key 粘贴进 Key 。

cmd 中输入

如果是第一次会提示是否 continue，输入 yes 后会显示： You've successfully authenticated, but GitHub does not provide shell access. 这就表示已经成功连接 GitHub 。
